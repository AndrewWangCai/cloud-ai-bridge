# 安全与隐私说明（用户向）

> 面向使用者的隐私/安全说明。漏洞报告与威胁模型见仓库根的 [SECURITY.md](../SECURITY.md)。

## 你的数据怎么被隔离
- **每个项目一个独立容器**：CPU/内存/进程数受限，`cap-drop=ALL`、禁止提权、专用网络。
- **绝不**使用 privileged / host 网络 / 挂载 docker.sock。
- 容器**禁止**访问云元数据地址、内网网段与 SMTP（防窃取云令牌/防滥用）。
- 你只能访问**自己**的项目；越权访问和路径穿越都有防护。

## 你的密钥怎么处理
- API Key 作为环境变量注入你的容器，**浏览器永远拿不到**服务端私有通道令牌。
- 日志做**脱敏**：不打印完整 Key/token。
- 当前原型中 API Key 以 0600 权限明文存于服务端 `data/`（生产建议叠加 KMS/加密）；**最稳妥是用会员号 OAuth 登录**，不长期托管 Key。

## 风控（防滥用，可能影响你）
- 同一 IP 每天注册有上限；异常 IP 可被封禁。
- 每用户并发连接数有上限。
- 容器**出站流量**超限会被自动停止 —— 不要在沙箱里跑爬虫/大量下载/代理。
- 单 workspace 磁盘**超配额**会被警告/停止。
- 触发以上限制时，项目会显示停止原因。

## 你应该做的
- **代码及时 push 到自己的仓库**，别依赖临时沙箱保存。
- 不要把高权限/长期密钥放进沙箱；优先用最小权限的 PAT、用完可吊销。
- 分享预览链接注意它**带 token**，拿到的人即可访问该端口页面（有过期）。

## 我们不做的事
- 不默认拿你的 GitHub token、不自动替你 push。
- 不跨用户共享你的代码或对话。
