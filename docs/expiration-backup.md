# 到期、备份、恢复与销毁

免费沙箱是**临时**的。理解生命周期，避免代码丢失。

## 生命周期一图流
```
新建 → 运行(RUNNING)
        │  默认 48 小时（公开测试；内测可配 5 天）
        ▼
   到期：停容器 + 自动备份(BACKED_UP)
        │  备份保留期（默认 10 天，可配）
        │  其间可：下载 / 恢复 / git push
        ▼
   超过保留期：彻底销毁(DESTROYED) —— workspace 和备份都删除
```

后台有定时清理任务（CleanupWorker），**即使你不打开网页**也会执行到期/销毁。

## 各状态含义
| 状态 | 含义 |
|---|---|
| RUNNING | 容器在跑，可用 |
| BACKED_UP | 已到期、已自动打包备份；可恢复/下载 |
| STOPPED | 被停止（如出站超限/超配额/管理员操作）|
| DESTROYED | 已彻底删除，不可恢复 |
| ERROR | 出错，可重试或联系管理员 |

## 我能做什么
在【配置】抽屉的数据按钮区：
- **💾 备份**：手动打包当前 workspace 到云端。
- **🔄 同步（恢复）**：从云端备份还原到 workspace（会重载终端）。
- **⬇️ 下载**：把备份 `.tar.gz` 下载到本地。
- **💥 销毁**：立刻彻底删除该项目（不可逆，谨慎）。

## 强烈建议
1. **到期前用 [Git](git-guide.md) 把代码 push 到你自己的仓库** —— 这是最可靠的保存方式。
2. 重要节点手动【备份】或【下载】一份。
3. 别把沙箱当长期存储；它是临时编程环境。

## 为什么我的项目「没了」
- 多半是**到期/超保留期被销毁**，或你点了销毁。
- 也可能只是 AI **没有上次的对话记忆**（文件其实还在）——文件看终端 `ls`，记忆是另一回事。
- 想让 AI 记住项目，用「项目说明」写进 `CLAUDE.md`（见 [ai-cli-guide.md](ai-cli-guide.md)）。

## 可配置项（管理员）
`FREE_RUN_HOURS`（默认 48）、`FREE_RUN_DAYS`、`RETENTION_DAYS`、`CLEANUP_INTERVAL_MS`。
